## Ethical Guidelines

### The Professional Pentester Code

1. **Always Get Authorization**
   - Written permission is non-negotiable
   - Verbal permission is NOT sufficient
   - Document everything

2. **Respect Scope**
   - Stay within authorized targets
   - Don't exceed agreed-upon testing methods
   - Stop immediately if you discover something unexpected

3. **Protect Data**
   - Don't access more data than necessary
   - Don't steal, copy, or exfiltrate data
   - Report data exposure responsibly
   - Delete test data after assessment

4. **Minimize Impact**
   - Use safe testing methods
   - Avoid denial of service
   - Don't damage systems
   - Test during approved time windows

5. **Responsible Disclosure**
   - Report vulnerabilities to the owner first
   - Give reasonable time to fix
   - Don't publicly disclose without permission
   - Follow coordinated disclosure processes

## Best Practices

### Before Testing

1. **Get Written Authorization**
   ```
   Required elements:
   - Specific systems/IPs authorized
   - Specific testing methods allowed
   - Time windows for testing
   - Contact information
   - Escalation procedures
   - Signatures from authorized parties
   ```

2. **Understand the Scope**
   - What systems are in scope?
   - What testing methods are allowed?
   - What is explicitly out of scope?
   - What are the time restrictions?

3. **Prepare Documentation**
   - Authorization letters
   - Scope agreements
   - Testing methodology
   - Reporting templates

### During Testing

1. **Stay in Scope**
   - Don't explore beyond authorization
   - Stop if you encounter out-of-scope systems
   - Document all activities

2. **Communicate**
   - Report critical findings immediately
   - Update stakeholders on progress
   - Escalate unexpected discoveries

3. **Be Careful**
   - Use safe testing methods
   - Avoid impacting availability
   - Don't access sensitive data unnecessarily

### After Testing

1. **Report Findings**
   - Detailed vulnerability reports
   - Clear remediation guidance
   - Risk prioritization
   - Proof of concept (safe)

2. **Clean Up**
   - Remove test accounts
   - Delete test data
   - Document cleanup activities

3. **Follow Up**
   - Verify fixes if requested
   - Provide additional guidance
   - Maintain confidentiality
